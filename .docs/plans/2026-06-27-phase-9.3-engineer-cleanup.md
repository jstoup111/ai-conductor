## Tasks

### Task 1 — A-3: remove the `gh!` non-null assertion (type-narrow the remote path)
- In `processIdea` step 4e, at the top of the `if (target.remote)` branch, capture
  `const gh = deps.gh;` and guard `if (!gh) throw new Error('engineer: remote target requires a gh runner')`.
- Replace `deps.gh!(...)` with `gh(...)`. The branch is now reached only via an explicit
  gh-present guard; no `gh!`.
- **RED:** a test where `target.remote` is set but `deps.gh` is undefined → `processIdea`'s
  per-idea catch prints the guard error and `ideasProcessed` stays 0 (no throw escapes the loop;
  no `gh` deref). And: a non-remote idea with `gh` undefined completes without dereferencing `gh`.
- **GREEN:** the guard above. `grep -n 'gh!' loop.ts` → 0 hits.

### Task 2 — A-1: strengthen routing repo-untouched assertions (falsifiable)
- Harden the create-offer test (~643) and decline test (~666): capture the proposed-repo
  directory listing AND registry record-count BEFORE the loop, assert byte-for-byte equality
  AFTER (`toEqual`), not merely that a string was printed. Keep the existing string assertion.
- Add a `registryCount` helper (read registry file → length, 0 if absent) in the test module.
- **Falsifiability check (dev-time, not committed):** temporarily inject a mutation (write a file
  into the proposed dir / append a registry record) and confirm each strengthened assertion FAILS;
  then remove the injection. Record in the commit body that falsification was demonstrated.
- No production change in this task.

### Task 3 — A-2: extract the post-authoring handoff into a named step
- New module `engine/engineer/handoff-step.ts` exporting
  `runHandoff(target, branch, deps): Promise<{ project: string }>` where `deps` carries
  `{ gh?, engineerDir?, launchFn?, print }`. It owns steps 4e (PR vs no-remote commit + print),
  4f (ensure-running fire-and-forget), and returns the authored entry for the caller to push.
- `processIdea` replaces lines 423-472 with: build handoff deps → `const entry = await runHandoff(...)`
  → `summary.authored!.push(entry); summary.ideasProcessed += 1;`. Reuses the Task-1 `gh` guard
  inside `runHandoff`.
- New focused unit test `test/engine/engineer/handoff-step.test.ts`: pr-opened path prints URL +
  records once + fires launch once; no-remote path commits-locally print + records once + fires
  launch once; launch failure is swallowed; pr-opened does NOT double-record (openSpecPr already records).
- **No behavior change:** full engineer acceptance suite stays green; `loop.ts` LOC drops materially.
