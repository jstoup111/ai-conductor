**Date:** 2026-06-25

## Summary

Adds a single-writer `registry.ts` module + two conduct-ts CLI subcommands (`register`, `create`)
+ a `/bootstrap` auto-register step. Atomic temp+rename writes, canonical-path dedup, credential
redaction, status provenance. ~13 tasks; new code in `src/conductor` + one SKILL.md + docs.

## Technical Approach

Per ADR-003, all writes funnel through `src/engine/registry.ts`: `resolveRegistryPath({home,env})`,
`readRegistry`, `upsertProject` (dedup by `realpath`, preserve `created` status), `writeRegistry`
(atomic `tmp`+`rename`), `redactRemote`, and types-only `ProjectRecord`/`RegistryReader`. Two CLI
subcommands in the conduct-ts CLI consume it; `skills/bootstrap/SKILL.md` gains a step that invokes
`conduct register`. Path resolution mirrors `user-config.ts` (injectable for tests). Git via execa.
**BUILD is dispatched per-task** (retro H-2/C-1) with a domain-review pass on each correctness
derivation (dedup, redaction, no-clobber).

- ADR-003 APPROVED (done). No migrations, no new deps.
