## Summary

Reworks the conduct-ts engineer from a Node REPL + `claude -p` subprocess authoring pipeline into an
**agent-hosted** loop whose TypeScript surface is deterministic primitives, while routing reasoning
and the **interactive, human-gated DECIDE clarity loops** run in the host agent. Adds a hexagonal
**intake port** (claude-session adapter only) and a **pidfile-`O_EXCL`** daemon liveness / 1-per-repo
mutex with fire-and-forget `ensure-running`, and fixes `launchDaemonDetached`. **38 tasks.**

## Technical Approach

**What already exists (from the superseded 9.3 cycle) and is reused or reworked:**
- `src/conductor/src/engine/engineer/routing.ts` — discriminated union `RoutingOutcome`
  (`confirmed|redirected|create|declined`) + `routeIdea`/`handleGateResponse`. **Reused**; the
  `RoutingProvider`/`claude` inference is **detached** from the spawn path (agent supplies reasoning;
  the union + exhaustive gate logic stay as deterministic helpers).
- `src/conductor/src/engine/engineer/target.ts` — `resolveTargetRepo` + `TargetPathMissingError`
  (canonical path, fail-fast). **Reused as the isolation foundation**; extended with the C1 write guard.
- `src/conductor/src/engine/engineer/handoff.ts` — `openSpecPr` (no-remote aware). **Reused** for FR-7.
- `src/conductor/src/engine/engineer/lesson-store.ts`, `governor.ts`, `flywheel-trend.ts`,
  `authored-ledger.ts` — flywheel read, FR-9 governor, FR-12 trend. **Reused**; verify against stories.
- `src/conductor/src/engine/daemon-backlog.ts` — `discoverBacklog` + `isStoriesApproved` +
  `planHasDependencyTree` + `isProcessed`. **FR-24 predicate already matches; verify-only, no change.**
- `src/conductor/src/engine/engineer/daemon-launch.ts` — `launchDaemonDetached`. **Bug fix** (FR-22):
  `args:['conduct','daemon','--project',project]` → spawn with `cwd: repoPath`, drop `--project`.

**What is removed (Option #1):**
- The Node readline REPL substrate in `engineer-cli.ts` `dispatchEngineer` (stdin readline) and the
  one-shot authoring in `engineer/authoring.ts` `authorSpec()` (subprocess/provider author run).
  `loop.ts` `runEngineerMode` is reshaped from "REPL that spawns" into the deterministic primitives
  the host agent calls (intake → route helpers → branch/commit/guard → handoff → ensure-running).

**What is new (additive TypeScript):**
- `engineer/intake/port.ts` (Envelope contract + `IntakePort` interface), `engineer/intake/claude-session.ts`
  (adapter), `engineer/intake/idempotency.ts` (source+sourceRef dedup) — ADR-009.
- `engineer/authoring-guard.ts` (path-prefix write guard) — C1/ADR-008.
- `engine/daemon-lock.ts` (the **only** module touching `.daemon/daemon.pid`: `acquire`/`isLive`/
  `reclaim`/`ensureRunning`) — ADR-010/C3.
- Regression-guard tests (no `claude -p`, no stub string, no DRAFT authored stories) — C2.

**Sequencing rationale:** primitives that others depend on first (Envelope/port, lock module), then
adapters and wiring (claude-session, ensure-running, launch fix), then the loop reshape that composes
them, then the cross-cutting guards/regression/verify tasks. Groups B and C are independent and can
proceed in parallel; Group A's loop reshape depends on both; Group D is verify-only.

## Prerequisites

- Vitest layout confirmed: tests under `src/conductor/test/engine/engineer/<module>.test.ts`,
  `test/engine/daemon*.test.ts`, `test/acceptance/engineer.test.ts`. Match `<module>.test.ts` naming.
- No migrations, no schema changes, no new runtime dependencies (`O_EXCL`/`process.kill` are stdlib).
