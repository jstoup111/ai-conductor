## Summary
Adds the github-issues async intake source + durable inbox queue + realized write-back to the
engineer, purely additively on the 9.3 port. **29 tasks** (Medium feature — in the 21–40 band;
surfaced for confirmation in §Scope).

## Technical Approach
All work lives under `src/conductor/src/engine/engineer/`. The seam is three small interfaces —
`IntakeSource` (capture), `IntakeQueue` (buffer), and the durable ledger — plus the github adapter
that implements `IntakeSource` + `IntakePort`. Build order is **inside-out**: contracts first, then
the ledger (the new dedup authority that replaces the in-memory guard), then the file-backed queue
(with its own `O_EXCL` claim primitive — never importing `daemon-lock.ts`), then the github adapter
(capture + write-back), and finally the loop/CLI wiring (poll-on-launch, `poll`, `forget`,
auto-reopen). `daemon-lock.ts` is never modified; the loop's existing `ensureRunning` import is
untouched, but the **queue module** carries a static no-import guard (C1). Removing
`intake/idempotency.ts` and repointing callers to the ledger (C2) happens early so nothing is built
on the doomed primitive. All `gh` access is injected (a `GhRunner`-style function) so unit tests
never hit the network; one integration test uses temp git repos + a fake gh for the full lifecycle
and the cross-repo isolation assertion (C3).

Data shapes:
- `.engineer/inbox/<id>.json` — one Envelope per file (claimable unit).
- `.engineer/ledger.json` — `{ "<source>\x00<sourceRef>": { status, branch?, prUrl?, attempts, capturedAt, lastSeenAt } }`.
- Ledger states extend the Envelope lifecycle with `claimed` and the terminal-ish `needs-manual`.

## Prerequisites
- Authenticated `gh` (runtime); for tests, an injected fake gh runner.
- 9.3 intake port/Envelope (present), 9.2 registry reader (present), `AuthoringGuard` (present).
