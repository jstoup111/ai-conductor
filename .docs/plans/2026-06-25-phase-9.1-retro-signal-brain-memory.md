## Summary

Adds a `brain-store` module + an emission step wired into the daemon's feature completion: on
daemon `done`/`halted`, assemble a structured signal (reusing report-renderer), produce a
narrative (Option A: emission owns it), and atomically append to `~/.ai-conductor/brain/`. Best-
effort and concurrency-safe. ~14 tasks, all in `src/conductor` + docs.

## Technical Approach

New `src/engine/brain-store.ts`: `resolveBrainDir()` (path + `$AI_CONDUCTOR_BRAIN_DIR` override +
auto-create), a `BrainSignal` type, `assembleSignal()` (parse the feature's `events.jsonl` + reuse
`report-renderer` aggregation + `FeatureOutcome`), `appendSignal()` (one atomic `O_APPEND` write),
and `writeNarrative()` (`narratives/<project>/<feature>-<runId>.md`). A `produceNarrative()` helper
generates the `done` narrative via the existing LLM provider (the work the in-loop retro did) and a
template-assembled short halt narrative for `halted` (no Claude call needed). `emitBrainSignal()`
wraps it all best-effort (log + swallow). Wired into `daemon-runner.ts makeRunFeature` **after
`readOutcome`, before `teardownWorktree`**, gated on daemon mode. The gate loop skips the in-loop
`retro` step under the daemon (manual runs unchanged). A types-only stub reader interface is
exported for 9.3.
