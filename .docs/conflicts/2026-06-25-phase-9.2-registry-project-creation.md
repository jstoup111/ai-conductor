## Degrading overlap (RESOLVED): FR-6 × ST-026 — create vs bootstrap scaffolding

**Stories:** "Create a new project (scaffold + register)" (FR-6) vs `ST-026` Project Detection and
Scaffolding (`/bootstrap` detects stack + generates CLAUDE.md + `.memory/`/`.docs/`).
**Type:** overlap · **Severity:** degrading

**Description:** Both produce a CLAUDE.md and scaffold. If `create` re-implemented bootstrap's
stack detection + onboarding, the two would drift and duplicate.

**Resolution applied:** `create` writes only a **minimal skeleton** (git init + template CLAUDE.md
+ `.gitignore` + register). Full onboarding stays in `/bootstrap`, run afterward, which (per ST-026)
skips-if-bootstrapped and fills gaps without overwriting, and (FR-8) auto-registers idempotently
(FR-4). So `create → bootstrap` composes without duplication. FR-6 story updated with this boundary
+ Done-When.

## Minor (RESOLVED): status provenance — `created` vs `registered`

`create` sets `status: created`; a later `/bootstrap` auto-register (FR-8) would set
`status: registered`, downgrading the provenance of a project we scaffolded. **Resolution:** the
register path must **not** clobber a `created` status into `registered` (preserve provenance, or
model `status` as a richer value). Flagged for architecture/ADR; FR-6 Done-When notes it.
