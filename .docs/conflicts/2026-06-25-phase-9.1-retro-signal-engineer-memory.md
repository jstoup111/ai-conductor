# Conflict Check: Phase 9.1 — Retro Signal + Engineer Memory Store

**Date:** 2026-06-25
**Stories checked:** `.docs/stories/phase-9.1-retro-signal-engineer-memory.md` (11) against existing
retro/daemon/conduct stories (`ST-024` dual-retrospective, `ST-010` feature-completion, `ST-005`
tier-based-skipping) and Phase 9 specs.
**Result:** **No blocking conflicts.** Two degrading interactions found and resolved in place.

---

## Degrading overlap (RESOLVED): FR-7 × ST-024 — retro output destination

**Stories:** "Daemon writes no retro into the project repo" (FR-7) vs `ST-024` Dual Retrospective
(the retro skill writes to `.docs/retros/`).
**Type:** overlap · **Severity:** degrading

**Description:** `ST-024`/the retro skill writes the narrative to the project repo's
`.docs/retros/`. FR-7 requires daemon-built features to write **no** retro into the repo. These
coexist only if the destination is **conditional on run mode**.

**Resolution applied:** The redirect is **daemon-only**. Manual `/conduct` runs preserve `ST-024`
unchanged (repo `.docs/retros/`); daemon runs route the narrative to the engineer store. FR-7's
negative path already asserts the manual-run behavior is unchanged. **Architecture decision
deferred to ADR:** whether the daemon (a) skips the in-loop retro step and generates the narrative
at emission time, or (b) lets the in-loop retro run and redirects its output to the store.
Recommendation: (a) — emission owns the narrative, simplest and avoids a repo write to undo.

---

## Degrading gap (RESOLVED): FR-5/6 × ST-005 — Small tier skips retro

**Stories:** "Done → full retro narrative" (FR-5) / "Halted → short halt narrative" (FR-6) vs
`ST-005` tier-based skipping (Small tier **skips** the retro step).
**Type:** state/sequencing gap · **Severity:** degrading

**Description:** A Small-tier daemon feature skips the retro step, so FR-5's "full retro narrative"
has no source. Emission must not assume a retro ran.

**Resolution applied:** `narrativeRef` is **optional**. A retro-skipped done feature still emits
the structured signal with `narrativeRef` absent/null; no narrative is fabricated. Updated: FR-5
story (new negative path + Done-When) and FR-3 story (schema: `narrativeRef` optional). Halted
features (FR-6) halt before retro anyway, so their **short halt narrative is generated at emission
time**, not from the retro step — already consistent.

## Concerns cleared

- **ST-010 feature-completion** ("commits uncommitted SHIP artifacts incl. `.docs/retros/`") — the
  daemon completion path (`daemon-runner`) is separate from the manual conduct completion; daemon

- **Emission point vs `FeatureOutcome`** — emission is additive/best-effort after `readOutcome`
  (FR-10); it does not alter `FeatureOutcome`. No conflict.
- **Internal** — FR-5 (narrative) + FR-8 (run-id versioning) are consistent (narratives keyed by
  `runId`); FR-3 empty-arrays + FR-9 metrics are consistent.

**Gate status:** Zero blocking conflicts. Two degrading interactions accepted + resolved in place
