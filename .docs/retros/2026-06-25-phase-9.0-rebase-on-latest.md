## Part A: Harness

- **H-1:** The CRITICAL FR-9 negative-path bug (in-progress rebase ‚Üí false "branch current" ‚Üí would ship a PR with live conflict markers) escaped TDD: the RED acceptance specs encoded FR-9's *happy* path (resume after a completed rebase) but not its *negative* path, even though the story (`stories/phase-9.0:281-290`) spelled it out. The 1015-test suite passed with the bug present; only the fresh-context evaluator caught it. **Gate that should have caught it:** `writing-system-tests` / the RED-spec step. **Severity:** High. **Fix:** make `writing-system-tests` mechanically derive at least one failing spec from *every* bullet under a story's `### Negative Paths`, and add a conduct post-spec gate: each negative-path bullet must map to a spec/test (mirrors the post-stories negative-path gate, one layer later).
- **H-2:** BUILD ran via direct agent delegation, not the conductor pipeline task-loop, so there was no `.pipeline/task-status.json` or `audit-trail/` ‚Äî the retro's own Data Collection had no structured gate/rework history; progress was reconstructable only from commits. **Severity:** Low. **Fix:** when BUILD is delegated (vs. the pipeline loop), have the delegation emit a minimal task-status/audit record, OR document that harness-self-development uses commit history as the audit trail so retro Data Collection doesn't expect the files.

**Proposed changes:**
