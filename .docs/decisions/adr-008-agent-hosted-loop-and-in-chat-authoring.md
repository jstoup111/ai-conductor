# ADR 008: Agent-hosted engineer loop + in-chat human-gated authoring (cross-repo isolation without a subprocess)

**Date:** 2026-06-26
**Status:** APPROVED
**Amended by:** ADR 2026-06-30 (engineer worktree authoring isolation) — adopts this ADR's
*deferred Option B* (author inside a dedicated worktree of the target) for the engineer, triggered
by **same-repo concurrency** (a force not weighed here). The canonical-path-no-cwd-fallback rule and
the path-prefix write guard from this ADR are **retained**; the worktree adds a second confinement
layer, it does not replace them.
**Deciders:** James (solo dev) + harness architecture-review
**Feature:** Phase 9.3 (redesign) — engineer loop, intake port & daemon liveness
**Supersedes:** ADR-004 (engineer authoring + cross-repo isolation), ADR-007 (interactive loop —
routing inference + spec-PR handoff)
**Decision surfaces:** execution model (FR-1/2), in-chat routing (FR-3/4/5), human-gated authoring
producing `Status: Accepted` artifacts (FR-6/7), cross-repo isolation without a process boundary
(FR-11)

## Context

The **shipped** 9.3 implemented the engineer as a **Node TTY REPL that spawns `claude -p`** to author
specs autonomously, writing **stub stories** (`# Stories: <idea>\n\n_Generated by engineer._`) that
are not `Status: Accepted`. That double-failed: it violated the human-driven gate (FR-6 intended
*orchestrating the DECIDE skills*, not autonomous authoring) **and** produced artifacts the daemon's
build-ready predicate skips (`isStoriesApproved` requires `Status: Accepted`, no DRAFT). The
spawned-subprocess substrate also produced the session-id / `captureOnly` / JSON-leak bug class.

ADR-004 and ADR-007 codified that substrate: **ADR-004** made cross-repo isolation (FR-11) a property
of the **OS process boundary** ("cwd-isolated subprocess authoring"); **ADR-007** described the loop
as an interactive REPL whose authoring crosses a process boundary. The redesign (handoff §3a) makes
the engineer **agent-hosted**: the loop, routing, and DECIDE all run **in the same chat session**,
human-gated, **with no spawned `claude`**. That removes the process boundary ADR-004 relied on — so
FR-11 must be re-established structurally by another means.

Forces:
- The whole point of the rework is *human-driven DECIDE in chat* — the operator is present and gates
  each step. A spawned autonomous authoring run is exactly what we are removing.
- **The defect is one-shot authoring, not merely the subprocess.** The shipped `ClaudeProvider`
  authoring path is a single inference call (idea in → artifacts out); even with the subprocess
  removed it cannot hold a **clarity loop** — brainstorm asking a question, the operator answering,
  requirements sharpening, then the next gate. Real DECIDE *is* that interactive refinement.
  Agent-hosted execution is the only model that delivers clarity loops; a Node-loop-plus-provider
  cannot — which is why the loop moves **into** the host agent rather than merely stopping the spawn.
- The DECIDE skills must produce **real `Status: Accepted`** PRD/stories/plan in the target repo, or
  the daemon will never build them.
- Without a subprocess, an in-process default that resolves to cwd is a latent cross-repo leak — the
  recurring Phase-9 bug ADR-004 designed out via the process boundary. We must not reintroduce it.
- The registry (ADR-003) already stores a canonicalized absolute path per project.

## Options Considered

### Execution model
- **A (chosen): Agent-hosted, in-chat, human-gated loop with clarity loops.** The host agent runs the
  loop, reasons about routing directly, and drives the real DECIDE skills in the chat **interactively**
  — brainstorm asks one question at a time, the operator answers, requirements refine across turns
  (the *clarity loop*), and each step is human-gated before the next. No Node TTY REPL, no `claude -p`.
  - *Pros:* restores human-driven DECIDE **with genuine clarity loops** (multi-turn requirement
    refinement, not a single inference); removes the spawned-subprocess bug class at the root;
    produces the same artifact shapes a normal DECIDE phase does; phone-drivable.
  - *Cons:* loses the OS process boundary as the isolation primitive (addressed below).
- **B: Keep the Node loop + `ClaudeProvider`, just stop spawning `claude -p`.** *Rejected* — the
  provider is a **one-shot** call, so authoring stays a pipeline with **no clarity loop**; it also
  keeps the autonomy framing. Removing the spawn alone does not restore interactive DECIDE.
- **C: Keep spawning `claude -p` but fix the prompts to emit `Status: Accepted`.** *Rejected* — keeps
  the autonomy violation and the session-id/JSON-leak bug class; merely papers over the symptom.

### Cross-repo isolation without a subprocess (FR-11)
- **A (chosen): Canonical-path resolution + absolute-path operation + a path-prefix write guard.**
  Resolve the target repo's canonical path from the registry (error if missing/stale — **no cwd
  fallback**, retained from ADR-004); all authoring operates on **absolute paths under that root**; a
  guard asserts every artifact write during an authoring run is path-prefixed under the resolved root,
  and authoring for A leaves sibling repo B and the engineer's own repo byte-unchanged.
  - *Pros:* structural (a guard, not vigilance); reuses ADR-003 canonical paths; no subprocess/worktree
    machinery; the operator-in-the-loop is a second check.
  - *Cons:* weaker than an OS boundary — relies on the guard + absolute-path discipline rather than
    the kernel refusing the write.
- **B: Author inside a dedicated git worktree/checkout of the target repo.** *Considered, deferred as
  escalation* — strongest confinement, but adds worktree lifecycle to every authoring run; unjustified
  at solo scale given the guard + human gate. Documented as the escalation if a leak is ever observed.

### Routing & PR handoff (retained from ADR-007)
- Routing inference (idea × registry → ranked candidates + rationale, human-gated) and **reuse of the
  existing PR machinery** are **retained unchanged**; only the *substrate* moves into the host agent.

## Decision

**Adopt agent-hosted, in-chat, human-gated execution (A) with isolation via canonical-path +
path-prefix confinement (A).** Supersede ADR-004 and ADR-007.

**Mechanism (locked):**
- **Loop (FR-1/2):** the host agent runs the long-lived loop; **no Node TTY REPL and no `claude -p`
  subprocess** is spawned for routing, authoring, or retro-narrative. A test asserts zero authoring
  subprocesses.
- **Routing (FR-3/4/5) — RETAINED from ADR-007:** the routing outcome is the discriminated union
  `{ kind: 'confirmed', project } | { kind: 'redirected', project } | { kind: 'create', name } |
  { kind: 'declined' }`; consumers switch **exhaustively** (no catch-all `default`); `declined`
  carries **no project**, so zero-writes-on-decline is **type-enforced**. Inference is the host
  agent's in-chat reasoning over the registry (no spawned `claude`). No-fit → offer 9.2 `create`;
  decline → no side effects.
- **Authoring (FR-6) — the fix:** run the **real DECIDE skills** (brainstorm→stories→plan) in chat,
  **interactively and human-gated, with clarity loops** — the operator may be asked clarifying
  questions, answer, and refine requirements across multiple turns before any artifact is accepted;
  each DECIDE step gates on operator approval before the next begins. Authoring targets the confirmed
  repo on a `spec/<slug>` branch and produces real **`Status: Accepted`** artifacts. **Never** a
  one-shot pipeline, **never** the stub form, **never** left at DRAFT, **never** via a spawned
  subprocess — each is a test failure (regression guards). The authoring **seam** the host agent fills
  is deterministic TypeScript (branch/commit/guard); the *reasoning and clarity loop* are the agent's,
  not a `ClaudeProvider` inference call.
- **Cross-repo isolation (FR-11):** resolve the `ProjectRecord` canonical `path`; **missing/stale →
  error before any write, no cwd fallback** (retained from ADR-004). All writes use absolute paths
  under that root; a **path-prefix guard** + a test assert authoring for A leaves B and the engineer's
  own repo byte-unchanged. Branch `spec/<slug>` off the target's default branch (derive via
  `git symbolic-ref refs/remotes/origin/HEAD`, never hardcode `main`); dirty tree not clobbered;
  existing branch suffixed.
- **PR handoff (FR-7) — RETAINED from ADR-007:** reuse the existing PR machinery against the target
  repo; report the URL; **no `gh pr merge`**, **no build** (ADR-005). No-remote (fresh local-only
  `create`) → commit on the branch + non-fatal PR-skip.
- **No build/merge path (FR-10):** unchanged and still structurally enforced by ADR-005.

## Consequences

### Positive
- Restores human-driven DECIDE **with clarity loops** (multi-turn requirement refinement, operator
  gated at each step); produces buildable `Status: Accepted` artifacts; eliminates the
  spawned-subprocess bug class; phone-drivable. The TypeScript surface shrinks to deterministic
  primitives (intake port, branch/commit/guard, PR handoff, lock, ensure-running, readers).
- Routing/PR decisions from ADR-007 and canonical-path resolution from ADR-004 are preserved.

### Negative
- **Cross-repo isolation is now guard-enforced, not kernel-enforced** (the key trade-off of going
  agent-hosted). Mitigated by canonical-path-only resolution (no cwd fallback), absolute-path
  operation, a path-prefix write guard + test, and the operator-in-the-loop. Escalation if a leak is
  ever observed: author inside a dedicated worktree of the target (Option B).
