## Feasibility

- **Stack compatibility — verified:** the change is pure TypeScript inside the
  existing conductor package. It requires no dependency, service, database, migration,
  authentication change, or infrastructure.
- **Provider identity seam — verified:** both `index.ts` and `daemon-cli.ts` hold the
  exact `llm_provider` key when they retrieve the provider. The policy can be resolved
  at those composition roots without changing `LLMProvider`.
- **Resolution seam — verified:** all engine model/effort values converge through
  `resolveStepConfig`; making policy input explicit gives one precedence-preserving
  boundary for step defaults and tier overrides.
- **Escalation seam — verified:** `escalateAttempt` is already pure and driven by two
  order arrays. Supplying arrays from a policy does not change retry-loop control flow.
- **Availability seam — verified:** `ModelAvailability` already accepts a ladder at
  construction. The configured ladder can continue to win; the policy only replaces
  the global default.
- **Documentation seam — verified:** the model table is generated from typed engine
  values. Switching its input to typed policies preserves the existing drift gate.
- **Worktree isolation — verified:** no ports, databases, processes, or shared runtime
  state are introduced. Policy objects are immutable process-local data.

## Alignment

- **Deterministic where possible:** provider selection and all default lookup remain
  deterministic engine machinery rather than prompt instructions.
- **Pattern consistency:** the design extends the existing typed single-source model
  metadata approach from
  `adr-2026-07-03-generated-model-table-single-source` instead of creating a parallel
  hand-maintained document.
- **Resilience alignment:** the design preserves the attempt-indexed escalation
  invariant from `adr-2026-07-05-retry-as-escalation-ladder` and the in-attempt
  availability walk from `adr-2026-07-03-reactive-model-fallback-ladder`.
- **Plugin boundary:** provider policy remains internal and does not expand the
  `LLMProvider` contract. Unknown plugins keep legacy behavior with a visible warning.
- **State management:** policies are immutable value objects; the only mutable state
  remains the existing per-process unavailable-model cache.
- **Security and data:** no new user input, persistence, authorization boundary, or
  sensitive data flow.
- **Diagram accuracy:** the registry overview and the updated escalation/fallback
  diagrams render successfully and describe the selected design.
- **Verify-claims verdict:** CLEAR. The operator approved the three-model per-step
  Codex policy, separate policy threading, and unknown-plugin compatibility behavior.

`conduct-ts overlap-scan` reported broad advisory overlap with numerous historical
unmerged `spec/*` branches on the conductor's common engine and documentation files.
The directly related `model-availability`, `retry-as-escalation`, and Fable policy
spec branches date from July 2–5 and describe behavior already present on `main`;
they are planning-history noise rather than competing implementations.

The active-looking `feat/codex-provider-900` worktree was checked separately. Its
feature landed on `main` as commit `1ef6aba7` (`#910`), and its stale branch does not
change the new policy module or the resolver/escalation/availability implementation
planned here. Its only candidate-path overlap is documentation and `CHANGELOG.md`.

**Advisory verdict:** no blocking dependency. Re-run the scan from the plan's exact
file union before BUILD because these shared engine files remain collision-prone.

## Risks

| Risk | Type | Likelihood | Impact | Mitigation |
|------|------|------------|--------|------------|
| A missed production resolver call silently uses the wrong provider | Integration | Medium | High | Make policy a required production dependency; enumerate and test every call site; as-built reachability sweep |
| Moving constants changes Claude behavior | Technical | Medium | High | Golden compatibility tests for every step, tier override, escalation rung, fallback rung, and precedence layer |
| Codex model entitlement differs by installation | Integration | Medium | Medium | Existing reactive `modelUnavailable` detection walks Sol → Terra → Luna and logs the downgrade |
| Unknown plugin providers receive unsuitable Claude defaults | Integration | Low | Medium | Preserve existing behavior, warn once, keep explicit overrides ahead of policy, document deferred plugin policy contract |
| Generated table conflates autonomous provider values with interactive skill pins | Technical | Medium | Medium | Provider-labelled engine columns; separately label interactive-only rows; retain Claude pin integrity checks |
